Icons

Each of the following icons is available in the /img folder, and can be linked to via the DLx CDN at https://cdn.digitallinguistics.io/img/{filename}. Some are available in multiple formats. Most icons are black by default, but for some there are versions in different colors as well, following the format {filename}.{color}.{ext}. For icons available in different sizes, the size is indicated by appending the dimensions to the filename, after an underscore, e.g. tunnel_64x64.png. DLx uses Material Design Icons for many of its icons.

  • Blog icon
  • The box icon is the logo for the DLx app
  • The browse icon is the logo for the DLx data explorer
  • Icon representing programming code, e.g. indicating an area with raw JSON data, etc.
  • The computer icon is used for areas concerning developers
  • The info icon is used for About pages or to indicate where additional information is available
  • A basic menu / navigation icon
  • This icon is used to represent the concept of a language
  • The GitHub Octicon logo
  • The question icon is used for questions/help pages
  • A generic tools icon
  • The tools icon is used as the logo for the Tools section of the DLx app
  • The JSON tunnel icon is the de facto logo for DLx
  • The blue wug is part of DLx branding, and can be used creatively wherever appropriate
  • The wugbot icon is another de facto logo for DLx, part of DLx branding, and can be used creatively wherever appropriate
  • The JSON tunnel icon as a favicon
  • The wugbot icon as a favicon

Patterns

abbr

Default styling for <code>&lt;abbr&gt;</code> tags, which should be used on the first mention of an abbreviation. It is often accompanied by a <code>&lt;dfn&gt;</code> tag in the same paragraph.

Example Markup

<p> <dfn>Digital Linguistics</dfn> (<abbr>DLx</abbr>) is the science of managing linguistic data, digitally. This is an example of a paragraph with an abbreviation tag. </p>

Live Example

Digital Linguistics (DLx) is the science of managing linguistic data, digitally. This is an example of a paragraph with an abbreviation tag.

bulleted

In DLx projects, lists are often used semantically rather than stylistically, e.g. a phrase includes a list of words. As such, lists have no styling by default, and list styling needs to be explicitly set with the <code>.bulleted</code> or <code>.numbered</code> styles.

Example Markup

<ul class=bulleted> <li>This is the first list item.</li> <li>This is the second list item.</li> <li>This is the third list item.</li> </ul>

Live Example

  • This is the first list item.
  • This is the second list item.
  • This is the third list item.

button

A basic button. This is applied automatically to buttons, but may also be applied using the .button class to any element.

Example Markup

dfn

A <code>&lt;dfn&gt;</code> tag is used the first time an element is defined. The definition of the term should be given in the surrounding text.

Example Markup

<p> <dfn>Digital Linguistics</dfn> (<abbr>DLx</abbr>) is the science of managing linguistic data, digitally. </p>

Live Example

Digital Linguistics (DLx) is the science of managing linguistic data, digitally.

field

Styles an element as a form field. This style is automatically applied to various form elements, such as the <code>&lt;input type=text&gt;</code> element.

Example Markup

<input class=field type=text value='This text input has been styled with the .field class'> <input class=field type=text disabled value='This text input is disabled'> <input class=field type=text pattern=aaa value='This text input is invalid'>

Live Example

fieldset

Used for grouping inputs together under a heading (<code>&lt;legend&gt;</code>). Remember that flexbox does not work well with <code>&lt;fieldset&gt;</code>, so you may need to wrap the items inside <code>&lt;fieldset&gt;</code> in a <code>&lt;div&gt;</code>.

Example Markup

<fieldset> <legend>Colors</legend> <label> <span>Blue</span> <input type=radio name=color> </label> <label> <span>Green</span> <input type=radio name=color> </label> <label> <span>Yellow</span> <input type=radio name=color> </label> </fieldset>

Live Example

link

The <code>.link</code> class is not automatically applied to anchor elements (<code>&lt;a&gt;</code>). It is, however, automatically applied to any anchor element that sits inside another inline element, such as <code>&lt;abbr&gt;</code> or <code>&lt;strong&gt;</code>, and to any element that sits inside an element whose content is usually inline, such as <code>&lt;p&gt;</code> or <code>&lt;h1&gt;</code>.

Example Markup

<a href=#> <p>This is a <code>&lt;p&gt;</code> tag wrapped in an <code>&lt;a&gt;</code> tag. It does not get link styling.</p> </a> <p> <a href=#>This is an <code>&lt;a&gt;</code> tag inside of a <code>&lt;p&gt;</code> tag. It gets link styling.</a> </p> <p> <a href="#">This</a> shows a link around a single word, inside a <code>&lt;p&gt;</code> tag. </p>

Live Example

tooltip

Adding the <code>.tooltip</code> class to any element will make a tooltip display on hover or when the element is focused. The content of the tooltip is contained in the <code>title</code> attribute for that element. Note that tooltips do not work on <code>&lt;input&gt;</code> elements - place the <code>.tooltip</code> class on the <code>&lt;label&gt;</code> instead.

Example Markup

<p> This is an example of an <span class=tooltip title='This is the inline tooltip.'>inline tooltip</span>. </p> <label class=tooltip title='This is a tooltip on a block element'> <span>Tooltip</span> <input type=text> </label>
