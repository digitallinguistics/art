Typography

Fira Sans Regular is the default font used throughout DLx tools, with a line height of 1.5. At larger sizes this font tends to look bold, in which case it may be a good idea to use Fira Sans Light (font-weight: 300;) instead.

Fira Sans Italic is used for italics.

Fira Sans SemiBold (not Fira Sans Bold) is used for bold styling.

Fira Sans SemiBold Italic is used for the combination of bold and italics.

Fira Sans Light (font-weight: 300;) may also be used for the default font, depending on what weight suits the page best. It provides a slimmer and slightly less bulky font profile, which is especially good for large headings.

Fira Sans Light Italic is the italicized version of Fira Sans Light.

Fira Mono is used for monospaced code blocks.

Fira Mono Bold is used for emphasized or bolded code.

Gentium Plus Regular is used for text that needs to support Unicode characters, such as data input fields.

Gentium Plus Italic is used for italicized text that requires Unicode support, such as inline citation of data.

Gentium Fonts are also enlarged slightly so that they can be used inline in stretches of Fira Sans text without too much of a size disparity.

Colors

The following colors are available as LESS variables and classes. Click on the color to copy it to the clipboard. To set the font color, use .{color} (e.g. .wugblue). To set the background color, use .{color}-background, (e.g. .wugblue-background).

@lightblue-transparent rgba(92, 175, 255, 0.2)
@verylightblue-transparent rgba(136, 196, 254, 0.2)

Icons

Each of the following icons is available in the /img folder, and can be linked to via the DLx CDN at https://cdn.digitallinguistics.io/img/{filename}. Some are available in multiple formats. Most icons are black by default, but for some there are versions in different colors as well, following the format {filename}.{color}.{ext}. For icons available in different sizes, the size is indicated by appending the dimensions to the filename, after an underscore, e.g. tunnel_64x64.png. DLx uses Material Design Icons for many of its icons.

  • The GitHub Octicon logo
  • The question icon is used for questions/help pages

  • The tools icon is used as the logo for the Tools section of the DLx app
  • The JSON tunnel icon is the de facto logo for DLx
  • The blue wug is part of DLx branding, and can be used creatively wherever appropriate
  • The wugbot icon is another de facto logo for DLx, part of DLx branding, and can be used creatively wherever appropriate

bulleted

In DLx projects, lists are often used semantically rather than stylistically, e.g. a phrase includes a list of words. As such, lists have no styling by default, and list styling needs to be explicitly set with the .bulleted or .numbered styles.

Example Markup


    <ul class=bulleted>
      <li>This is the first list item.</li>
      <li>This is the second list item.</li>
      <li>This is the third list item.</li>
    </ul>

field

    <input class=field type=text pattern=aaa value='This text input is invalid'>

link

The .link class is not automatically applied to anchor elements (<a>). It is, however, automatically applied to any anchor element that sits inside another inline element, such as <abbr> or <strong>, and to any element that sits inside an element whose content is usually inline, such as <p> or <h1>.

Example Markup


    <a href=#>
      <p>This is a <code>&lt;p&gt;</code> tag wrapped in an <code>&lt;a&gt;</code> tag. It does not have link styling.</p>
    </a>
    
    <p>
      <a class=link href=#>This is an <code>&lt;a&gt;</code> tag inside of a <code>&lt;p&gt;</code> tag. It gets link styling.</a>
    </p>
    
    <p>
      <a class=link href="#">This</a> shows a link around a single word, inside a <code>&lt;p&gt;</code> tag.
    </p>

Live Example
