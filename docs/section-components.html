6 Components

Discrete, self-contained pieces of UI. These styles are not applied automatically, but rather need to be applied using the class with the same name as the element (e.g. apply the .button class to all <button> elements.)

6.1 Banner

A site banner at the top of the page. When including additional elements inside the banner, you might want to set margin-left: auto; on the element to push it to the opposite end of the banner from the <h1>.

Example

DLx Pattern Library

<header class=banner>
  <h1>DLx Pattern Library</h1>
</header>

6.2 Bulleted List

In DLx projects, lists are often used semantically rather than stylistically, e.g. a phrase includes a list of words. As such, lists have no styling by default, and list styling needs to be explicitly set with the .bulleted or .numbered styles. List items with text should include a <p> tag or other relevant element within them, following the principles of semantic markup.

Example

This is text leading up to the bulleted list (for reference).

  • This is the first list item.
  • This is the second list item.
  • This is the third list item.
<p>This is text leading up to the bulleted list (for reference).</p>
<ul class=bulleted>
  <li>This is the first list item.</li>
  <li>This is the second list item.</li>
  <li>This is the third list item.</li>
</ul>

  pattern=[0-9]

6.8 Pre

Used for standalone code blocks (for inline code blocks, see the <code> tag). If the <pre> element is used by itself, without a <code> element inside it, it will simply display as preformatted text. If a <code> element is included inside the <pre>, however, that element will be display: block;.

Example

This is a <pre> element without a <code> element inside of it.
  This is a <pre> element with a <code> element inside of it.

It is displayed as a standalone block of code.

{
  "dateCreated": "2017-03-13T02:36:22.123Z",
  "dateModified": "2017-03-13T04:51:26.134Z",
  "name": {
    "eng": "Chitimacha",
    "ctm": "Sitimaxa"
  }
}

<pre class=pre>This is a &lt;pre&gt; element without a &lt;code&gt; element inside of it.</pre>

<pre class=pre>
  <code>This is a &lt;pre&gt; element with a &lt;code&gt; element inside of it.

It is displayed as a standalone block of code.

{
  "dateCreated": "2017-03-13T02:36:22.123Z",
  "dateModified": "2017-03-13T04:51:26.134Z",
  "name": {
    "eng": "Chitimacha",
    "ctm": "Sitimaxa"
  }
}</code>
</pre>

6.9 Reference List

Styling for a bibliographic reference list. Note that each reference should be an <li> tag containing a <p> tag. NOTE: You may have to further customize the styling of the <hX> element depending on which header you are using. Use <cite> for publication titles.

Example

Reference List

This is a paragraph giving a short description of this reference list.

  1. Hall, Patrick J. 2014. Serialization and hierarchy: From data to corpus in linguistic fieldwork. M.A. thesis. Linguistics, University of California, Santa Barbara.

  2. Hieber, Daniel W. 2015. Review: Endangered languages & new technologies (Jones). Language Documentation & Conservation 9: 344–350.

<section class=reference-list>

  <h2>Reference List</h2>

  <p>This is a paragraph giving a short description of this reference list.</p>

  <ol>

    <li>
      <p>Hall, Patrick J. 2014. <cite>Serialization and hierarchy: From data to corpus in linguistic fieldwork</cite>. M.A. thesis. Linguistics, University of California, Santa Barbara.</p>
    </li>

    <li>
      <p>Hieber, Daniel W. 2015. Review: Endangered languages &amp; new technologies (Jones). <cite>Language Documentation &amp; Conservation</cite> 9: 344–350.</p>
    </li>

  </ol>

</section>

6.10 Text Line

A text input styled with a simple underline rather than an inset box. A <p> tag may also be included immediately following the <input> element in order to provide a brief description, explanation, or other help text below the input. NOTE: The :not() selector causes problems for KSS, so that the :disabled variant below isn't rendering correctly; however, it does render correctly in production.

Examples

Default styling

:disabled

For disabled text inputs

:focus

For focused text inputs

:invalid

For invalid data in text inputs

<label class="textline {{modifier_class}}">
  <span>Label Name</span>
  <div>
    <input
      class="{{modifier_class}}"
      inputmode=verbatim
      placeholder='placeholder data'
      type=text
    >
    <p>An abbreviation for this language variety. May contain only letters and numbers.</p>
  </div>
</label>

<label class="textline {{modifier_class}}">
  <span>Label Name</span>
  <div>
    <input
      class="{{modifier_class}}"
      inputmode=verbatim
      pattern=[0-9]
      placeholder='placeholder data'
      type=text
      value='Invalid data'
    >
    <p>An abbreviation for this language variety. May contain only letters and numbers.</p>
  </div>
</label>
